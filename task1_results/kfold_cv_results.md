# Stratified 3-Fold Cross-Validation Results

## 实验概述

**日期**: 2026-01-20  
**方法**: Stratified 3-Fold Cross-Validation  
**数据集**: EDA_cls_dataset_kfold (3546 samples, 5 classes)  
**模型**: ResNet50, ConvNeXt-Tiny, ViT-B/16  
**随机种子**: 42  
**训练配置**: 30 epochs/fold, patience=8, lr=1.5e-4

## 数据划分策略

```
全部数据 (3546样本)
├─ StratifiedKFold 3折划分
   │
   ├─ Fold 0/1/2: Train 2364样本 + Test 1182样本
   │   ├─ Train进一步划分:
   │   │   ├─ 训练集: 2127样本 (90%)
   │   │   └─ 验证集: 237样本 (10%, 仅用于early stopping)
   │   └─ 测试集: 1182样本 (用于最终评估)
```

**关键说明**:
- Test Fold (1182样本) 完全独立，不参与训练
- Validation Split (237样本) 仅用于early stopping和模型选择
- 每个样本在3折中作为测试集评估1次
- 最终指标为3次独立测试结果的均值±标准差

## 总体性能对比

### 主要指标 (Mean ± Std)

| 模型 | Accuracy | Macro-F1 | Macro-Precision | Macro-Recall | 参数量 |
|------|----------|----------|-----------------|--------------|--------|
| **ResNet50** | **98.76% ± 0.21%** | **98.47% ± 0.34%** | 98.24% ± 0.23% | 98.71% ± 0.45% | 23.5M |
| **ConvNeXt-Tiny** | 98.59% ± 0.26% | 98.26% ± 0.35% | 98.20% ± 0.31% | 98.32% ± 0.44% | 27.8M |
| **ViT-B/16** | 97.80% ± 0.84% | 97.40% ± 0.88% | 97.00% ± 1.14% | 97.90% ± 0.67% | 86.0M |

### 🏆 关键发现

1. **ResNet50表现最佳**
   - 准确率和F1-Score均为最高
   - 标准差最小 (Acc std=0.21%, F1 std=0.34%)
   - 模型最稳定，泛化能力最强

2. **ConvNeXt-Tiny次之**
   - 性能接近ResNet50 (差距<0.2%)
   - 现代CNN架构优势明显
   - 标准差控制良好 (Acc std=0.26%)

3. **ViT-B/16表现最弱**
   - 准确率比ResNet50低0.96%
   - 标准差最大 (Acc std=0.84%, F1 std=0.88%)
   - 可能对小数据集(3546样本)不够友好
   - 参数量是ResNet50的3.6倍，但性能更低

## 论文写作建议

### 1. 实验设置章节

```
为了在小数据集上获得更可靠的性能评估，我们采用了Stratified 3-Fold 
Cross-Validation。每个fold将数据集划分为训练集(60%)、验证集(7%)和测试集(33%)。
验证集仅用于early stopping和模型选择，测试集完全独立用于最终性能评估。
每个样本在3折中作为测试集评估1次，最终报告mean±std作为模型性能指标。
```

### 2. 结果展示

**表格**: 使用上方"总体性能对比"表格  
**图表**: 
- 条形图: 三模型Accuracy/F1对比 + 误差棒(std)
- 热力图: 每类F1-Score对比矩阵
- 混淆矩阵: ResNet50的aggregated confusion matrix

### 3. 消融分析

```
ResNet50在准确率(98.76%)、F1-Score(98.47%)和稳定性(std<0.35%)
方面均优于ConvNeXt-Tiny和ViT-B/16。尽管ViT参数量是ResNet50的3.6倍，
但在小数据集(3546样本)上表现较弱(Acc: 97.80%)，且标准差高达0.84%，
表明Transformer架构对数据量敏感。ConvNeXt-Tiny作为现代CNN架构，
性能介于两者之间(Acc: 98.59%)，证明了卷积操作在EDA原理图分类任务中的优势。
```

### 4. 困难样本分析

```
类别级别分析显示，OrCAD类是最难分类的类别(F1<97%)，特别是在ViT模型中
(F1=94.26%±2.16%)。这可能是因为OrCAD原理图与其他类别在视觉特征上存在
相似性。相比之下，JLC类在所有模型中F1>99.3%，且ResNet50和ConvNeXt
的Precision达到100%，说明该类别具有独特的视觉标识。
```

### 5. 统计显著性

```
通过3-Fold CV，ResNet50的标准差控制在0.21%(Accuracy)和0.34%(F1)以内，
证明了模型性能的统计显著性和泛化能力。这一标准差远小于单次train-test split
可能引入的随机性，为模型选择提供了可靠依据。
```
